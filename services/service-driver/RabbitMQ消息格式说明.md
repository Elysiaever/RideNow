# RabbitMQ 消息传递格式说明

### JSON 消息格式

#### 完整示例
```json
{
  "driverId": 1001,
  "lng": 116.404,
  "lat": 39.915
}
```

#### 字段说明

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `driverId` | Long | 是 | 司机唯一标识ID | `1001` |
| `lng` | Double | 是 | 司机当前位置经度（WGS84坐标系） | `116.404` |
| `lat` | Double | 是 | 司机当前位置纬度（WGS84坐标系） | `39.915` |

#### 字段详细说明

**1. driverId（司机ID）**
- **数据类型**：`Long`（长整型）
- **取值范围**：正整数，与数据库中司机表主键对应
- **业务含义**：唯一标识一个司机，用于在 Redis GEO 中存储和检索司机位置
- **示例**：
  - `1001` - 司机ID为1001的司机
  - `2567` - 司机ID为2567的司机

**2. lng（经度）**
- **数据类型**：`Double`（双精度浮点数）
- **取值范围**：-180.0 ~ 180.0
- **坐标系**：WGS84（World Geodetic System 1984）
- **精度建议**：保留6位小数（精确到约0.1米）
- **业务含义**：司机当前所在位置的经度坐标
- **示例**：
  - `116.404` - 北京天安门附近
  - `121.473` - 上海外滩附近
  - `-73.935` - 纽约时代广场附近

**3. lat（纬度）**
- **数据类型**：`Double`（双精度浮点数）
- **取值范围**：-90.0 ~ 90.0
- **坐标系**：WGS84
- **精度建议**：保留6位小数（精确到约0.1米）
- **业务含义**：司机当前所在位置的纬度坐标
- **示例**：
  - `39.915` - 北京天安门附近
  - `31.230` - 上海外滩附近
  - `40.758` - 纽约时代广场附近
